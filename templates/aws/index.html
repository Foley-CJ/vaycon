NoSQL Databases Analysis: Twitter Utilization


NoSQL databases have been gaining popularity ever since they first appeared in 1998. Since that point, multiple offerings have entered the space each with a slightly different value proposition. Given the uniqueness of each offering, it can be difficult to detemine which product to deploy. One dimension of that comparision is the level of support and community surrounding the offering.

In this analysis we investigate that specific dimension by evaluating how each of three of the most popular NoSQL databases utilize Twitter. More specifically, we evalaute MongoDB, DynamoDB, and Cassandra by pulling data from the Twitter API and investigating it with python's plotly package.


High Level Product Summary


To start this analysis we begin by reviewing a few key facts about the three products. These key facts include when the database was first made available, when it joined Twitter, the number of followers, total tweets sent, and tweets sent within 2021.

Database Initial Release Twitter Date Followers Total Tweets 2021 Tweets
Cassandra 2008 2009 40.7K 479 117
DynamoDB 2012 2008 16.5K 6.3K 1180
MongoDB 2009 2008 381.5K 20.8K 1601

These figures reveal that tenure nor the quantity of tweets alone can predict the number of followers: the key performance indicator that we're interested in exploring. However this data does reveal that both DynamoDB and Cassandra have taken, they do suggest that Cassandra is more active on twitter than it has been historically. In order to reveal any insights deeper analysis is required.

    Results: 2021 Analysis


    There are several key takeways as well as additional opporutnities for furthe ranalysis surroindign this investigation. The three major takeways freom the analysis include:

    • MongoDB is effective at drawing engagement in the form of likes as well as growing its follower base by utilizing question based tweets.
    • Utilizing live events such as MongoDB.live or MondoDB.local in conjuction with twitter works well to build a platform and extend the reach of that platform. AWS could utilizing something similiar during it's AWS re:Invent conference
    • Since September of this year DynamoDBs tweets have significantly dropped off allowing other communities to grow further.

    Some additional opporutnites and outstanding questions that should still be addressed as as follows:

    • Does feature/product enahancement announcements draw a large engagement from twitter followers?
    • Does the sentiment or style of communciations have a significant impact on the effectiveness of twitter
    • Are there altenrative social media pplications that could provide a more valuabe return on the amount of time invested?
    • Does the twitter community represent an important part of the actual database community?
